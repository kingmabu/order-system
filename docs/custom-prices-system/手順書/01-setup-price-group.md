## ステップ3：新しいファイルを追加してスクリプトを貼り付ける（2分）

> 🛑 **重要：`Code.gs` を絶対に上書きしないでください**
>
> 開発用 Client Information の `Code.gs` には、本番からコピーされた以下の**本番ロジック**が入っています：
> - **CustomerID 自動生成**（3桁ID・ID_Map隠しシート管理）
> - **顧客名でソート**
> - **QBO顧客リンク自動生成**（F列）
> - **クライアントフォーム自動生成**（"Client Form"テンプレート）
> - **メニュー追加・onEditトリガー**（B/D/E列編集時の自動処理、A列入力時のAccessKey生成）
>
> 上書きすると業務が壊れます。本手順では `Code.gs` の隣に**新規ファイルを追加**して、その中にこの設定スクリプトを貼り付けます（プロジェクトルール「**既存スクリプトのロジック変更禁止・追加のみOK**」に準拠）。

### 操作

1. Apps Script エディタの左サイドバー、**ファイル**ヘッダーの横にある **`＋`** ボタンをクリック
2. **スクリプト** を選択
3. ファイル名を `setup_price_group` と入力して **Enter**
   - ⚠ 同じ機能のファイル（例：`set_up_price_group.gs` など類似名）が**既に存在する場合は、まず先にそちらを削除**してください。同一の定数名・関数名が複数ファイルにあると `Identifier has already been declared` エラーで実行できません。
4. 新しい `setup_price_group.gs` が開きます（中身は `function myFunction() {}` の空関数だけ）
5. エディタ内のコードを**すべて削除**します（`Ctrl + A` → `Delete`）
6. 以下のローカルファイルを開き、中身をすべてコピーして貼り付けます：
   ```
   apps-script\client-info-dev\setup_price_group.js
   ```
7. 右上の **保存ボタン（フロッピーマーク）** をクリック、または `Ctrl + S` で保存します
8. プロジェクト名を聞かれた場合は元のまま（変更不要）

## 困ったときは

### エラー：`SyntaxError: Identifier 'PG_VALID_GROUPS' has already been declared`
- 別の `.gs` ファイル（例：`set_up_price_group.gs` など類似名）に同じ定数・関数が定義されています。
- 古い方を**削除**してください（左サイドバーでファイル名の右の `⋮` → **削除**）。
- このスクリプトは**1ファイルだけ**にしてください。

### エラー：「シート "Client list" が見つかりません」
- シート名が違う可能性。タブ名の前後の半角スペースや大文字小文字を確認してください。

### 警告：「既に23列目以降に何か入っています。上書きされます。」
- W列以降に既にデータがある場合の警告です。
- **W列="Price Group" / X列="Markup %"** であれば**正常**（再実行のため）、そのまま続行してOK。
- それ以外の列構成の場合は**実行を止めて Claude に報告**してください。

### エラー：「権限が必要です」が消えない
- ブラウザのキャッシュをクリアして再ログインしてください。

### 「実行」ボタンが押せない
- スクリプトに構文エラーがある可能性。エディタ下部に赤字が出ていないか確認してください。

### 関数選択プルダウンに `setupPriceGroupColumns` が出ない
- 左サイドバーで **`setup_price_group.gs`** が選択されているか確認してください（`Code.gs` を選んでいると本番ロジックの関数しか出ません）。
- ファイルを切り替えると、関数選択プルダウンの候補も切り替わります。

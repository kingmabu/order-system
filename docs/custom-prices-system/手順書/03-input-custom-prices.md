# 手順書3：Custom Prices 75レコードを入力する

このドキュメントは、開発用 Cost list の **Custom Prices シート**に、Individual 分類15社の個別単価を**「価格管理」メニュー経由**で入力するための作業手順書です。

- **想定所要時間**：**2〜3時間**（1件あたり 1〜2分 × 75件）
- **入力件数**：約 **75レコード**（15社 × 平均5アイテム）
- **対象スプレッドシート**：開発用 Cost list（ID: `1NPCw-Bz0kokXEe2Tv2PcF9XkQsa1c5IqiGh3IIkWpV8`）

---

## 0. 事前準備チェック（5分）

以下が**すべてOK**になるまで、本作業は開始しないでください。

- [ ] **手順書1完了**：Client list の W/X列が追加されている
- [ ] **手順書2完了**：Client list の W列にIndividual 15社が入力されている
- [ ] **Cost list 側 Apps Script デプロイ済み**：開発用 Cost list を開くと、メニューに **「価格管理」** が表示される
  - もし「価格管理」メニューが**見えない**場合 → Claude に「手順書3の事前準備：Cost list の Apps Script デプロイ手順がほしい」と依頼してください
- [ ] **QBO Pricing Rules のスクリーンショット / 一覧**：Individual 15社の Per Item 価格情報が手元にある

## ステップ1：入力用の一覧表を準備する（30〜60分）

フォームに入力する前に、**QBO から Individual 15社の per-item 価格をすべて書き出した一覧表**を準備します。これがないと、フォーム入力中に QBO と何度も切り替えることになり時間がかかります。

### 1-1. 推奨フォーマット（Excelまたはメモ帳）

| Customer ID | Customer Name | SKU | Item Name | Custom Price | Note |
|---|---|---|---|---|---|
| 011 | BENI HOLLYWOOD | B033 | Beef Short Rib | 22.50 | 初期移行 |
| 011 | BENI HOLLYWOOD | C037 | Drumsticks | 18.00 | 初期移行 |
| 035 | KUSHIYAKI BAR | B008 | Beef Ribeye 10oz | 14.20 | 初期移行 |
| ... | ... | ... | ... | ... | ... |

**作り方：**
1. QBO の Pricing Rules 一覧を開く
2. Individual 15社それぞれの Per Item 価格設定を確認
3. 上記フォーマットで Excel / メモ帳に書き出す
4. Customer ID は **3桁ゼロ埋め**（例：`11` ではなく `011`）

### 1-2. 単位の確認

各 SKU の **量り売り/定量売り** を Item List（開発用ID: `1dIiwCvK8DRXiRX9jGcaKmlc_x6QxsVmS_0dGpukjXAY`）で確認します。

| Item List I列 | 価格の単位 |
|---|---|
| ☐（チェックなし） | **ポンド単価**（$/lb） |
| ☑（チェック済み） | **箱単価**（$/unit） |

> ⚠ QBOで設定されている価格が「ポンド単価」か「箱単価」か、必ず確認してください。**間違えると価格が桁違いになります。**

### チェックポイント

- [ ] 一覧表に**約75行**ある
- [ ] Customer ID がすべて **3桁ゼロ埋め**
- [ ] SKU がすべて記入されている
- [ ] Custom Price がすべて**正の数値**（0や空欄なし）
- [ ] 単位を確認済み（ポンド単価 / 箱単価）

## ステップ3：1件目を試しに入力する（5分・動作確認）

いきなり75件続けて入力する前に、**1件目だけ入れて挙動を確認**します。

### 3-1. フォーム入力

1. **Customer**：プルダウンから最初の顧客を選択（例：`BENI HOLLYWOOD (011)`）
2. **SKU**：プルダウンから商品を選択（例：`B033 - Beef Short Rib`）
3. **Custom Price**：数値を入力（例：`22.50`）
4. **Note**：`初期移行` と入力
5. **追加** ボタンをクリック

### 3-2. 結果を確認

フォームに **「追加完了」** などの成功メッセージが出たら、Custom Prices シートを開いて確認：

1. メニュー **価格管理 → 一覧を確認** をクリック
2. Custom Prices シートに**1行追加されている**ことを確認：

| A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|
| 011 | BENI HOLLYWOOD | B033 | Beef Short Rib | 22.5 | 5/15/2026 | 初期移行 |

- [ ] A列：3桁ゼロ埋め（`011`）
- [ ] B列：顧客名が自動入力されている
- [ ] D列：商品名が自動入力されている
- [ ] F列：本日の日付が自動入力されている

### 3-3. ログ確認

`Custom Price Log` シートタブをクリックして確認：

| Timestamp | Action | Customer ID | Customer Name | SKU | Item Name | Price Before | Price After | Note |
|---|---|---|---|---|---|---|---|---|
| 2026/05/15 14:30:15 | Add | 011 | BENI HOLLYWOOD | B033 | Beef Short Rib |  | 22.5 | 初期移行 |

- [ ] Timestamp が自動入力されている
- [ ] Action が `Add`
- [ ] Price Before が **空欄**（追加なので変更前はない）
- [ ] Price After に値が入っている

### チェックポイント

- [ ] 1件目の入力が正常に動いた
- [ ] Custom Prices シート + Custom Price Log シート の両方に記録された

> ⚠ もしエラーが出る場合は、**75件入れる前にここで止めて Claude に報告**してください。

## ステップ4：残り74件を入力（90〜120分）

1件目がOKなら、残りを淡々と入力します。

### 4-1. 効率化のコツ

- **同じ顧客の複数SKUは続けて入力**：Customer ID は同じなので、SKUと価格だけ変えればよい
- **顧客順に並べて入力**：顧客切り替えが少なく済む
- **15件ごとに小休止**：集中力維持
- **5件ごとに Custom Prices シートで件数確認**：入力漏れ早期発見

### 4-2. 重複エラーが出た場合

「既に登録があります」エラーが出たら、**同じCustomer ID + SKU が既に入っている**ということです。

- **本当に重複している場合**：その行はスキップ
- **値を変えたい場合**：いったんフォームを閉じて **価格管理 → 個別価格を変更** で変更
- **入力ミスの場合**：Customer ID / SKU を確認して入れ直す

### 4-3. 入力途中で中断する場合

- フォームを閉じてOK（途中までの入力は Custom Prices シートに残っている）
- 再開時は「個別価格を追加」を再度起動

### チェックポイント

- [ ] 入力後、Custom Prices シートに **約75行**ある
- [ ] Customer Price Log シートに **Add 行が同数**ある

## ステップ5：最終確認（10分）

### 5-1. 件数チェック

Custom Prices シートの末尾の空きセル（例：I1）に貼り付け：

```
=COUNTA(A2:A1000)
```

→ 約 **75**（実際の入力件数）になっていればOK。

### 5-2. 顧客カバレッジチェック

Custom Prices シートの A列でユニークな Customer ID を確認：

1. A列を選択
2. メニュー **データ → ピボットテーブル**（または `=COUNTA(UNIQUE(A2:A1000))` で件数確認）
3. ユニーク値が **15** であることを確認 → Individual 15社全員がカバーされている

### 5-3. データ品質チェック

- [ ] 全行で Customer ID が **3桁ゼロ埋め**
- [ ] 全行で Custom Price が **正の数値**（0や空欄なし）
- [ ] 全行で Update Date が**日付として認識**されている（左寄せの文字列になっていない）
- [ ] 重複なし：A列 + C列の組み合わせがすべてユニーク

### 5-4. Custom Price Log の整合性

- [ ] Custom Price Log シートの行数 ≧ Custom Prices シートの行数（変更や削除があれば多くなる）
- [ ] すべての `Add` アクションが Custom Prices シートと一致

### 5-5. 確認式を削除

I1 などに入れたチェック式を削除します。

## 困ったときは

### Customerプルダウンに顧客が出ない / 15社未満
- 手順書2の入力が反映されていない可能性。
- フォームを閉じて、ブラウザを完全リロード（`Ctrl + Shift + R`）してから再起動。
- それでも出ない場合は Client list の W列を再確認。

### 「既に登録があります」エラーが頻発する
- 一覧表に重複がある可能性。Excel上でCustomer ID + SKU の重複チェックをしてください。

### フォームが固まる / 反応しない
- ブラウザのモーダルダイアログを閉じて再起動。
- それでもダメなら **ブラウザを完全リロード**（`Ctrl + Shift + R`）。

### 入力した価格を間違えた
- **価格管理 → 個別価格を変更** から修正可能。
- 変更履歴は Custom Price Log に自動記録されます。

### 間違って違う顧客に入れてしまった
- **価格管理 → 個別価格を削除** で削除 → 再度追加。
- ログには削除と追加の両方が残ります（履歴として正常）。

### Customer Name の自動入力がおかしい
- フォームは Client list のB列を顧客名として参照しています。
- B列以外に顧客名がある場合は、Claude に「Cost list の `CL_COL_CUSTOMER_NAME` を調整してほしい」と依頼してください。
